        # sqlite — Что такое SQLite

        Homework-шаблон для урока **l1_what_is_sqlite** (Что такое SQLite) на платформе Vibe Learn.

        ## Что делать

        Реализуй мини-«формат файла приложения» на Go с modernc.org/sqlite (чистый Go, без CGO).
Программа открывает app.db, создаёт схему notes(id, title, body, created_at) если её нет,
и предоставляет функции AddNote / GetNote / ListNotes / SearchByTitle. Сравни старт и
поиск с baseline-реализацией на одном большом JSON-файле (заглушка в lib). Тесты гоняют
на :memory:-базе: проверяют CRUD, что схема идемпотентно создаётся, и что поиск
использует индекс (через EXPLAIN QUERY PLAN).

## Контекст (из transfer-задачи урока)

Ты пишешь десктопное приложение для заметок (как Obsidian/Notion-локально). Сейчас
каждая заметка — отдельный markdown-файл, метаданные (теги, даты, связи) — в большом
JSON-файле, который приложение целиком читает и переписывает при каждом изменении.
С ростом до 50k заметок: старт медленный (парсинг гигантского JSON), поиск по тегам
линейный, при крэше во время записи JSON бьётся.

**Вопрос:** как SQLite чинит каждую из трёх болей? Опиши:
(a) что положишь в SQLite, а что оставишь в markdown-файлах (и почему именно так);
(b) как пропадает «медленный старт» и «линейный поиск по тегам»;
(c) почему крэш во время записи больше не бьёт данные;
(d) и честно: есть ли тут сценарий, где SQLite НЕ помог бы (подумай про синк между
    устройствами).

## Recap из урока

- SQLite — **embedded SQL-движок**: библиотека в твоём процессе, вся БД в одном файле, без сервера/сети/конфига. Самая распространённая БД в мире.
- Ментальная модель: **SQLite конкурирует с `fopen()`, а не с PostgreSQL** — заменяет самописные файловые форматы, давая SQL+транзакции+индексы бесплатно.
- Микросекундная латентность — потому что запрос это **вызов функции в процессе**, читающий страницы файла; нет сериализации и сетевого round-trip'a.
- Выигрывает: формат файла приложения, edge/mobile, тесты (`:memory:`), read-heavy single-server (с WAL — вполне прод).
- Не подходит: высокая конкурентная запись, доступ по сети с многих машин, данные больше одного диска, per-user права. Туда — PostgreSQL.

        ## Как работать

        1. Платформа Vibe Learn создаёт копию этого репо в твоём GitHub-аккаунте по клику «Начать домашку» на странице урока (через GitHub `/generate`, codecrafters-pattern).

        3. CI (`.github/workflows/ci.yml`) запускает `go vet` + `go test ./...` на каждый push. Платформа слушает результат через webhook от GitHub Actions и обновляет статус домашки на странице урока.

        ## Локальное окружение

        - Go 1.22+
        - SQLite встроена — **никакого сервера и docker-compose**. БД это один файл (`DATABASE_PATH`) или `:memory:` в тестах. Драйвер `modernc.org/sqlite` — чистый Go, без CGO, так что CI собирается без компилятора C.

        ## Запуск

        ```bash
        # Прогнать тесты (бегут на :memory:-базе, ничего поднимать не нужно)
        go test ./...

        # Запустить main (создаёт схему, печатает marker; замени stub на реализацию)
        go run .

        # На файловой БД (нужно для WAL/concurrency-уроков):
        DATABASE_PATH=./app.db go run .
        ```
